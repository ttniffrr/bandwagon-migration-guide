## 什么叫"机房迁移"：先说清楚这个功能本身

搬瓦工的机房迁移（Datacenter Migration）是套自助系统，在 KiwiVM 控制面板里操作，免费、不限次数。迁移完成后 IP 地址会变，但 VPS 里的数据、SSH 密码、安装的程序全部保留，大概等个十几分钟到几小时，完成后会发邮件通知。

简单说：机房变了，IP 跟着换，VPS 里的东西原封不动。

这个功能本来是给用户调整线路用的——电信用户发现当前机房晚高峰丢包，切一个 CN2 GIA 机房试试；移动用户觉得延迟高，换个软银或 CMIN2 看看。就这个用途。

不过，这里有两个"触发不能变机房"的硬条件。

## 不能变机房的原因一：IP 被墙了

这是绝大多数人碰到的情况。

搬瓦工有一条明确政策：**一旦检测到你的 VPS IP 在国内被完全屏蔽，Migrate to another DC 功能自动禁用。** 不是卡顿，是直接封掉这个入口。

为什么这样设计？机房迁移会换 IP，有人就想用这个办法绕过 IP 被封——白嫖一个新 IP。搬瓦工不允许这个操作，所以直接从根源堵死。

操蛋的地方在于：系统不会明确告诉你"你的 IP 被墙了，所以不让迁移"。你看到的只是迁移请求一直转，或者报一个错误代码，你以为是系统故障，实际上是 IP 状态的问题。

### 怎么确认 IP 是不是真的被墙了

先登录 KiwiVM 控制面板，然后打开搬瓦工官方 IP 检测工具（路径在 KiwiVM 菜单里，Test Network 或者 IP Blacklist Check 之类的选项），点 **Test Main IP**。

结果有两种：
- **IP NOT BLOCKED**：国内能正常访问，IP 没问题，不能迁移的原因在别处
- **IP BLOCKED**：IP 被墙了，换机房功能被锁，这就是根本原因

注意：一定要先登录 KiwiVM 再去开检测页面，否则会提示 session timed out，检测结果不准。

## 不能变机房的原因二：套餐本身锁定了机房

这个原因比较好理解，但新手容易踩坑，因为买的时候没注意看描述。

搬瓦工部分套餐在定价时就绑定了特定机房，价格里包含了这个机房的资源，不允许迁移。典型的是香港 CN2 GIA 套餐和日本东京 CN2 GIA 套餐——买这两个本来就是冲着这两个地方的顶级线路去的，如果允许随便换到其他机房，那套餐定价逻辑就乱了。

怎么判断？回到购买时的套餐详情页，看有没有"Location: Hong Kong Only"或类似的机房限定说明。后台迁移列表里如果只显示当前机房、没有其他选项，也是同样的意思。

说实话这种情况处理起来没什么好的出路，套餐锁死就是锁死的，想换线路只能新买一个可以迁移的套餐。

## 怎么排查：一步步来，别急

按这个顺序检查，能省掉很多白折腾：

1. **先确认 VPS 是不是单纯临时故障**：登录 KiwiVM 看 VPS 状态，搬瓦工有服务器状态页面，看看当前机房有没有在维护
2. **用官方工具检测 IP**：按上面说的步骤，结果是 BLOCKED 还是 NOT BLOCKED，先搞清楚
3. **看 VPS 状态是否正常运行**：如果 VPS 本身就是 Stopped 或者 Suspended，先处理这个
4. **检查套餐是否支持迁移**：KiwiVM 里打开迁移页面，看可选机房列表里有没有其他选项

排查完，基本能定位到是哪个原因。

## IP 被墙之后怎么办：唯一官方出路

换机房的路堵死了，只剩一个官方渠道：**付费换 IP**。

在 KiwiVM 后台找到换 IP 的申请入口，提交申请后系统会生成一个账单，付钱确认，搬瓦工会给你换一个新的、干净的 IP 地址。新 IP 保证在黑名单里没有，全球连通性正常。整个过程基本是秒换，但有时需要人工处理，最多等 24 小时。

价格方面，具体以申请页面显示为准，价格会有波动，不是固定的。

几个注意点：
- 换 IP 之后你的原 IP 会被回收
- 不要随便点，选对需要换 IP 的那台 VPS
- 付钱后如果没有立刻换成功，等几小时，不要反复提交

不想付钱怎么办？理论上也可以等——IP 被封有时候会在一两个月后自动解封，但也可能更久，看运气。如果你的 VPS 已经年付，扔掉重买太亏，就只能走付费换 IP 这条路了。

## 如果 IP 没被封，但迁移还是失败：常见错误代码

这种情况通常是临时性的，不是账户问题：

- **Region is full**：目标机房满了，等几小时再试，或者换个机房试
- **Migration backend not available**：后台系统临时不可用，等一等再操作
- **Failed to migrate**：可能是目标机房暂时不接受该套餐，换个机房或者等官方修复

这几个报错不需要联系客服，等待就行。如果一直出现同样的错误超过 48 小时，再考虑提工单。

## 几个容易踩的坑，顺手说一下

**换机房会消耗流量。** 这个很多人不知道。迁移过程需要把 VPS 里的数据传到新机房，消耗的流量大概是 VPS 已用硬盘大小的 4-8 倍。比如 VPS 用了 5GB 硬盘，一次迁移可能消耗 20-40GB 流量。不是不限次数不代表可以随便玩，流量是会被扣的。

**换回原机房 IP 可能不变。** 比如你从 DC6 切到 DC9，再切回 DC6，大概率 IP 还是原来那个（除非原 IP 被别人占了）。这个特性有时候有用——比如你想测试一下新机房，不满意还能切回去，IP 不变。

**IP 被封和退款。** 官方政策明确：因 IP 被封导致无法使用的情况，不支持退款。买之前要清楚这一点。如果你用的场景有 IP 被封的风险，建议先用月付测试，稳了再换年付。

## FAQ

**Q：搬瓦工换机房要钱吗？**

不收费，迁移本身是免费的，但会消耗月流量配额，每次消耗量视 VPS 里的数据大小而定。

**Q：换机房之后我的数据还在吗？**

在的。迁移只影响 IP 和物理位置，VPS 里的数据、系统、密码、配置文件全部保留。

**Q：IP 被封之后，等一段时间会自动解封吗？**

有可能，但时间不确定，可能几周，可能几个月，也可能更长。如果急着用，只能走付费换 IP 的路。

**Q：IP 没被墙，套餐也支持迁移，但后台怎么都选不了目标机房怎么办？**

先检查是不是目标机房满了（Region is full），换一个其他机房试试。如果所有机房都无法选，等几小时后重试，或者提工单。

**Q：我买的是限量版套餐，能换机房吗？**

不一定，要看具体套餐说明。有些限量版是固定机房，有些可以换。在 KiwiVM 的迁移页面能看到实际可选项，有选项就能换，没有就换不了。
